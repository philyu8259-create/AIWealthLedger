# 国际版真机联调清单

适用构建：`APP_FLAVOR=intl`

## 0. 联调前置

### 必备环境变量
先复制模板：

```bash
cp .env.example .env
```

至少确认以下项目：

```bash
GEMINI_API_KEY=
GOOGLE_VISION_API_KEY=
FINNHUB_API_KEY=
GOOGLE_IOS_CLIENT_ID=
GOOGLE_SERVER_CLIENT_ID=
GOOGLE_IOS_REVERSED_CLIENT_ID=
```

### 启动前自检
- [ ] `flutter pub get`
- [ ] `flutter analyze`
- [ ] `ios/Runner/.env` 与项目根目录 `.env` 内容一致，避免再次误读旧值
- [ ] 真机已登录 Apple ID（用于 Apple Sign In）
- [ ] iOS provisioning / signing 正常

### 推荐启动命令
```bash
flutter run -d <DEVICE_ID> --debug --dart-define=APP_FLAVOR=intl
```

## 1. 启动与基础外观
- [ ] App 可正常启动，不闪退、不卡 logo
- [ ] 桌面名为 `AI Wealth Ledger`
- [ ] 系统权限弹窗为英文
- [ ] 首页/设置页主要文案为英文，无明显中文残留

## 2. 国际登录

### Email 登录
- [ ] Welcome 页可进入 Email 登录
- [ ] 输入合法邮箱后可进入 app
- [ ] 退出后再次进入，登录态可恢复

### Google 登录
- [ ] 点击 Google 登录可拉起 Google 授权
- [ ] 授权完成后能正确回跳 app
- [ ] 登录后用户态正常落地
- [ ] 重启 app 后登录态仍在

### Apple 登录
- [ ] 点击 Apple 登录可拉起系统授权
- [ ] 首次授权成功后能进入 app
- [ ] 再次登录表现正常
- [ ] 取消授权 / 无 Apple ID 时错误提示清晰

### 登录后恢复链路
- [ ] `_postSignInBootstrap()` 正常执行
- [ ] 账单恢复无异常
- [ ] 股票持仓恢复无异常
- [ ] VIP restore / sync 无明显报错

---

## 3. 首页 AI 文本记账

### 有 key 场景
前提：`GEMINI_API_KEY` 已配置
- [ ] 输入英文消费文本可成功解析
- [ ] 解析结果分类、金额、备注基本合理
- [ ] 可确认保存到账单

建议样例：
- `coffee 4.5`
- `uber ride 18.2`
- `salary 3200`

### 缺 key 场景
前提：清空 `GEMINI_API_KEY`
- [ ] 点击 AI 解析时直接提示未配置 Gemini
- [ ] 不会伪装调用国内 provider
- [ ] 不会出现假死或无反馈

---

## 4. OCR 拍照记账

### 有 key 场景
前提：`GOOGLE_VISION_API_KEY` + `GEMINI_API_KEY` 已配置
- [ ] 可拉起拍照 / 相册选择
- [ ] OCR 能返回识别结果
- [ ] AI 能把 OCR 文本解析成账单
- [ ] 保存后账单列表可见

### 缺 key 场景
前提：清空 `GOOGLE_VISION_API_KEY` 或 `GEMINI_API_KEY`
- [ ] 进入 OCR 前直接提示未配置
- [ ] 不会拍完照后才失败
- [ ] 提示文案明确指出缺的是哪类能力

---

## 5. AI Insights / Prediction

### 有 key 场景
前提：`GEMINI_API_KEY` 已配置，且已有一定账单数据
- [ ] Prediction 页可正常加载
- [ ] insight / warning 为英文
- [ ] 无明显中文残留

### 缺 key 场景
前提：清空 `GEMINI_API_KEY`
- [ ] Prediction 页直接提示未配置 Gemini
- [ ] 不再只显示模糊的 failed

---

## 6. 资产页 / 美股

### 无 key 场景
前提：`FINNHUB_API_KEY` 为空
- [ ] 设置 / 资产相关提示明确显示美股 provider 未配置
- [ ] 资产页不会误打旧 A 股数据源
- [ ] 新增美股 / 刷新行情时给出明确提示

### 有 key 场景
前提：`FINNHUB_API_KEY` 已配置
- [ ] 可搜索美股代码或名称
- [ ] 无结果时有空状态提示
- [ ] 搜索失败时有错误提示
- [ ] 可新增美股持仓
- [ ] 手动刷新可拿到最新行情
- [ ] latest price / change / market value 正常显示
- [ ] 美股数量规则为整股，不再按 A 股 100 股校验

建议样例：
- `AAPL`
- `TSLA`
- `NVDA`
